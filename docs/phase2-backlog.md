# Phase 2 Backlog

## Priority A (after 1-2 pilot groups)

1. Quiz mode per topic
   - Single-choice and short-answer checks.
   - Show answers only in lecturer mode.
2. Topic-level feedback
   - Quick buttons: "clear", "hard", "need examples".
   - Export feedback summary after session.
3. Search across all modules
   - Fast search by keyword, interview question, code sample.

## Priority B (stability and scale)

1. Content validation script
   - Validate required fields in topic files.
   - Fail build if schema is broken.
2. Module packs
   - Keep modules in separate files and lazy-load them.
3. Session presets
   - 45, 60, 90-minute presets with topic priority filters.

## Priority C (engagement)

1. Lightweight animations for transitions
   - Keep under 200ms, no heavy effects.
2. Presenter confidence widgets
   - "Next likely question" suggestions from topic data.
3. Export handout to PDF
   - Student mode handout without lecturer notes.

## Optional desktop packaging

- Keep web app as source of truth.
- Add Tauri packaging only after web flow is stable.
